# Parity gates — the verification contract

PyTorch (Serenity / ai-toolkit / diffusers) is the source of truth. Every
trainer-pipeline unit is gated against a reference oracle generated by the
reference's OWN runtime. No circular comparisons; no synthetic loss scales.

This document is the *what to check* companion to `TENETS.md` (the *why*). It
records the gate sequence, the oracle infrastructure, and the worked, MEASURED
ideogram4 vertical.

---

## Oracle infrastructure

- **Reference venv** (the oracle's runtime): `/home/alex/serenityflow-v2/.venv`
  (torch 2.10 + cu128, diffusers, transformers). Run offline: `HF_HUB_OFFLINE=1`.
- **Oracle scripts** live next to the Mojo gates under
  `serenitymojo/models/.../parity/*.py`. They are DEV TOOLS — never imported by
  shipped Mojo. Each dumps a `.safetensors` fixture + a `_meta.json`.
- **Staged loading (24 GB rule):** load ONE big model at a time
  (VAE → free → text encoder → free → transformer). Freeing means deleting EVERY
  lingering reference (a leaked `lm`/`ie`/`hs`/`pe`/`cm` keeps the 11 GB Qwen
  resident and OOMs the next model). Set `PYTORCH_ALLOC_CONF=expandable_segments:True`.
  GPU-only — no CPU offload of activations.
- **Follow the production dtype** (TENET 6): the oracle runs the component in the
  same dtype the reference ships (e.g. the ideogram4 transformer in bf16).

## Running a gate (Mojo side — the main loop owns it)

```bash
cd /home/alex/mojodiffusion && rm -f serenitymojo.mojopkg && \
  pixi run mojo run -I . -I /home/alex/serenity-trainer/src <smoke.mojo>
```

`rm -f serenitymojo.mojopkg` forces a clean build (only ONE `mojo` builds at a
time — never race it). Read the actual cosine, not "it printed something."
Metric: `ParityHarness(0.999).compare(...)` → cosine; also watch `max_abs`. For
loops, gate **per-step** (cosine is accumulation-blind).

## Gate sequence (per model vertical)

1. **Encoders** — VAE encode (image → latent) + text encode (caption → features)
   vs the reference, cos ≥ 0.999. Tokenizer ids exact vs the reference tokenizer.
2. **Predict** — packed inputs (byte-exact) + `predict_velocity` (the training
   forward at LoRA B=0) vs the reference predict, cos ≥ 0.999, on a REAL sample.
3. **B=0 split == monolithic** — the split training forward
   (embed → LoRA blocks → final) must equal the monolithic inference forward at
   B=0, cos ≥ 0.999. (TENET 4 — the load-bearing gate.)
4. **Train step** — one `predict → MSE flow loss → backward → AdamW` on real
   data: loss in the reference's range, LoRA-B 0 → nonzero on ALL adapters, 0
   nonfinite.
5. **Real LoRA file** — load a REAL reference LoRA: key names + shapes + rank +
   dtype match (ai-toolkit ideogram4: `diffusion_model.layers.*.lora_A/B.weight`).
6. **Sampler** — denoise from a fixed `latent0` to `latent_final` (per-step
   cosine), then VAE decode → a valid image matching the reference (PSNR).

## Worked vertical: Ideogram-4 (MEASURED 2026-06-07)

Reference: ai-toolkit `ideogram4/{ideogram4.py, src/pipeline.py}` + ideogram4-ref
`src/ideogram4/`. Real data: `/home/alex/1/datasets/gigerver3_json` (Giger LoRA,
70 imgs + structured-`.json` captions; config `config/gigerver3_ideogram4_lora.yaml`).

| Gate | Smoke / probe | Result |
|---|---|---|
| VAE encoder (img→128 latent, mean) | `serenitymojo/.../parity/ideogram4_vae_encode_probe.mojo` | mean **0.99996**, latents **0.99996** |
| Text encoder (Qwen3-VL 13-tap) | `serenitymojo/models/dit/parity/chunk7_qwen_probe.mojo` | **0.99999** |
| Tokenizer (`.json` caption → ids) | `serenitymojo/tokenizer/ideogram4_tokenizer_probe.mojo` | 15/15 **exact** |
| Predict (real giger) | `smoke/ideogram4_predict_gate.mojo` | packing **exact**, velocity **0.9999373** |
| **B=0 split == monolithic** | `smoke/ideogram4_lora_b0_gate.mojo` | velocity **0.9999373** (== predict), loss **0.96100146** |
| Train step (1 step, real giger) | `smoke/ideogram4_lora_train_step_gate.mojo` | loss **0.96100146**, adapter_b_l1 0→**3147.6** |
| Real ai-toolkit LoRA load | `smoke/ideogram4_real_lora_file_parity.mojo` | 408 tensors / **204 adapters** / rank 32 / BF16 |
| Sampler (8-step CFG-7) | `serenitymojo/pipeline/ideogram4_pipeline.mojo` | final_z **0.96** / decoded **0.99** (bf16 ceiling) |

**Reference loss cross-check:** torch `MSE(velocity, target) = 0.961231` on the
giger sample ≈ Mojo train-step loss `0.96100146` (the ~2e-4 gap is the bf16
velocity error, consistent with velocity cos 0.99994). In Serenity's 0.2–1.0
range for an untrained base on a new style.

### Pipeline files (what each gate exercises)

- `model/Ideogram4VAE.mojo` — VAE encode/decode seams (borrows
  `serenitymojo/models/vae/ldm_encoder.mojo::load_ideogram4_vae_encoder`).
- `model/Ideogram4TextEncoder.mojo` — Qwen3-VL 13-tap (borrows
  `serenitymojo/models/text_encoder/ideogram_qwen3vl.mojo`).
- `model/Ideogram4Predict.mojo` — packing + `predict_velocity` + add_noise + target.
- `model/Ideogram4LoRABlock.mojo` — fp8 block LoRA fwd/bwd + the 34-block stack.
- `trainer/Ideogram4LoRATrainStep.mojo` — embed(frozen) → stack LoRA → final(frozen)
  → MSE → backward → AdamW.
- `modelSampler/Ideogram4SampleLoop.mojo` — the inference denoise driver.
- `modelLoader/Ideogram4LoRALoader.mojo` — real ai-toolkit LoRA format.

### Known limits (honest)

- The train step trains the **204 block adapters** (34×6). The **7 global
  targets** (input_proj / llm_cond / t_emb.mlp_in,out / adaln_proj /
  final_layer.adaln_modulation,linear) are NOT yet wired.
- The sampler matches at the IMAGE level (decoded 0.99 / PSNR ~29.7). The final_z
  0.96 is DiT-INTERNAL bf16-kernel divergence vs the bf16 reference, amplified
  over the CFG loop (single forward = 0.99994). MEASURED: RNE on the per-step
  boundary cast is byte-identical → not the boundary; tightening it needs F32 DiT
  compute (and the reference is bf16, so the benefit is unproven).
- The end-to-end dataLoader cache + train-loop driver are owned/in-progress
  elsewhere; gate them on a real giger batch when they land.
